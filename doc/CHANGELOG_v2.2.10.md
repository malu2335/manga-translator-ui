# v2.2.10 更新日志

发布日期：2026-06-26

## ✨ 新功能

### Qt UI：界面定义集中管理

- 桌面端 Qt UI 代码统一迁移到 `desktop_qt_ui/ui/`，按 `main_page`、`editor`、`secondary_pages`、`widgets`、`icons` 分类管理。
- 主页面、编辑器、二级页面编辑器、通用控件和图标资源不再分散在 `main_view_parts`、`widgets`、`styles`、`utils` 等多个目录中，后续修改界面时可以集中在 `ui` 目录内定位。
- 移除旧 UI 路径的兼容转发文件，避免同一套界面定义在多个目录之间绕转，降低维护成本。

### Qt UI：样式与主题集中化

- 新增统一样式入口 `desktop_qt_ui/ui/styles.py`，收拢主页面、二级页面、按钮、表格、输入框、菜单、提示框等常用 QSS 定义。
- 新增 `desktop_qt_ui/ui/theme.py` 和 `desktop_qt_ui/ui/theme_tokens.py`，集中维护主题运行时、调色板和主题 token。
- 设置页布局文件 `settings_tab_layout.json` 从 `desktop_qt_ui/locales/` 迁移到 `desktop_qt_ui/ui/main_page/`，多语言文本和 UI 布局定义不再混放。

## ⚡ 优化

### Qt UI：二级页面视觉一致性

- 统一自定义 API 参数编辑器、提示词管理编辑器、AI 上色提示词编辑器、替换规则编辑器等二级页面的背景、输入框、按钮、表格和弹窗样式。
- 浅色主题背景改为纯白，避免浅蓝底色和主页面风格不一致。
- 去除侧边栏和设置页中多余的灰色分隔条，减少界面噪声。
- 下拉菜单、滚动区域、二级页面容器改为跟随主题 token，避免深色/浅色主题切换后背景仍残留固定黑色或透明异常。

### Qt UI：按钮状态统一

- 统一主要操作按钮、描边按钮、危险操作按钮和小型工具按钮的主题样式。
- 「开始翻译」等主操作按钮恢复为蓝色主题按钮，避免出现白底白字或只有蓝色描边的问题。
- 提示词编辑器和 AI 上色提示词编辑器中的删除按钮统一使用红色危险按钮样式。

### 打包资源整理

- Release 打包流程改为复制 `desktop_qt_ui/ui` 目录，确保迁移后的图标、主题样式和设置页布局 JSON 在打包版本中可用。
- 保留 `desktop_qt_ui/locales` 作为多语言文本目录，避免影响 Web/Qt 共用的翻译资源。

### 默认配置

- 默认配置模板中的 OCR 设置更新为 `paddleocr_vl`，新用户和重新生成配置时会优先使用新的默认 OCR 方案。

## 🐛 修复

- 修复提示词编辑器中父级容器局部 `setStyleSheet("background: transparent;")` 截断样式继承，导致两个删除按钮仍显示为白色的问题。
- 修复提示词管理编辑器内局部 `QTabWidget` 样式覆盖统一按钮样式的问题，词表/术语区域的删除按钮现在会正确显示为红色。
- 修复设置页布局 JSON 迁移后开发环境和打包环境路径不一致的问题。
- 修复替换规则编辑器和文件夹选择器在 UI 目录迁移后仍按旧目录层级推算资源路径的问题。
- 修复行距倍率小于 1.0 时文本渲染行距计算不准确的问题。
- 修复加入 API 轮询后 Sakura 在 API 管理页不再显示地址和词典路径配置的问题；Sakura 保持固定兼容密钥，不暴露自定义 Key 或轮询槽。
- 修复 PaddleOCR-VL 偶发重复符号幻觉输出的问题：检测到大量重复字符/符号串时会自动使用更严格的生成参数重新推理，多次重试仍异常则丢弃该结果。
- 修复字体管理预览在同一字体族的不同字重之间无法区分的问题；预览现在直接按字体文件路径生成图片，避免 Qt 字体族合并导致 Medium/Bold/Heavy 切换无效，并降低切换卡顿。
- 修复在 API 管理面板中修改渲染器、翻译器等配置后，切换到设置页或编辑器时界面未同步更新的问题；现在切换到设置页会重新加载配置项，切换到 API 管理页会刷新 API 分组，进入编辑器会重新填充属性面板的下拉选项。
- 修复 API 管理面板中多个输入框连续输入时，因共用单个防抖定时器导致前面的修改丢失的问题；改为暂存所有待保存修改并统一刷盘，输入框失去焦点或按回车时立即保存，测试接口、获取模型列表、切换预设和开始翻译前强制保存所有待保存的 Key。
- 修复 API 管理面板图片类接口测试图过小的问题；测试用 PNG 从 2 像素调整为 50 像素，避免部分图片 API 拒绝过小输入。
- 修复开启气泡英文排版时，渲染阶段尝试写入只读的 `TextBlock.horizontal` 属性导致任务中断的问题；现在会通过文本方向字段强制横排。

## 📝 说明

- 本版本重点是桌面 Qt UI 目录结构和样式系统整理，为后续维护主页面、编辑器和二级页面 UI 提供统一入口。
- 旧的 `desktop_qt_ui/locales/settings_tab_layout.json` 路径已废弃；设置页布局现在维护在 `desktop_qt_ui/ui/main_page/settings_tab_layout.json`。
