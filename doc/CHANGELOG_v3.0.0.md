# v3.0.0 更新日志

发布日期：2026-06-28

## 🐛 修复与改进

### 编辑器底图标志保留修复
- 修复 Qt 桌面端编辑器导出时重写 JSON 会丢失 `upscale_ratio` / `colorizer` 标志的问题。现在编辑器保存区域数据时会保留已有的上色/超分底图来源信息，避免再次打开图片时将仍有效的 `editor_base` 误判为过期并删除。

### 设置页性能优化
- 优化 Qt 桌面端设置页切换性能。现在设置页会缓存已构建的控件，普通点击或切回设置页时不再反复销毁并重建全部设置项；配置导入、语言切换等需要刷新 UI 的场景仍会正常重建。

### 编辑器画笔光标优先级修复
- 修复 Qt 桌面端编辑器中彩色画笔/擦除工具悬停到已选中文本框控制柄时，鼠标光标会被移动或缩放光标抢占的问题。现在蒙版笔、蒙版橡皮、彩色画笔和彩色擦除统一遵循绘制工具光标优先。

### 修复中韩文横排断句
- 修复韩文横排自动断句按字符切分的问题。现在韩文（`KOR` / `ko` / `ko_KR`）会优先在单词之间的空格处换行，只有单个词本身超过文本框宽度时才退回到逐字兜底拆分，提升韩文翻译文本的可读性。
- 修复中文目标语言 `CHS` / `CHT` 未进入 CJK 横排断句器的问题，避免误用英文断句逻辑。
- 完善 CJK 收尾标点禁则：当 `，。！？` 等标点本身触发换行宽度限制时，会优先贴回上一行，避免出现标点位于下一行开头的情况。

### AI 渲染器文本替换修复
- 修复了在使用 AI 渲染器（OpenAI / Gemini）时，`text_replacements.yaml` 中配置的文本替换规则（如通用/横排/竖排符号或文本正则替换）不生效的问题。现在 AI 渲染模块在组装翻译 Prompt 前会正确加载并应用该规则。

### AI 渲染器前置画框与坐标去除（极简参考原文渲染 + 彻底擦除原文防重叠）
- 彻底去除了 AI 渲染器在发送的前置图像上绘制红色编号框与数字的逻辑，直接发送干净的去字/原图背景。
- 移除了冗长且大模型不易精确理解的绝对像素坐标，改为极简提示词，让模型在渲染时根据“原文（original text）”参考定位气泡。
- **强制擦除原文防重叠**：优化了默认提示词，明确要求大模型在渲染翻译前，必须**完全擦除原图上对应的日语文本**，防止翻译后的文字与原日语文字在气泡中重叠或并存。同时，也会自动识别并干净擦除所有残留的日语注音假名（Furigana/Rubi 字符）以及漏检文本，并将其转换为翻译渲染，保证整页漫画画面百分之百干净。

### 修复异常文本误判导致 API 密钥被误拉黑的问题
- 修复了当 AI 渲染失败抛出通用 `RuntimeError` 异常时，其文本描述中包含 "HTTP 404" 的排查细节信息会被轮询系统（`api_key_rotation.py`）的正则表达式抓取，误判定为发生了真实的 HTTP 404 永久网络错误，从而导致该 API Key 被错误的拉黑屏蔽。现在系统在提取状态码时会自动过滤并跳过 `RuntimeError`、`ValueError` 等内置异常类型的正则匹配，避免发生误拉黑。
- API 通道状态现在会区分不同 API Key，更换 Key 后不会继续继承旧 Key 的本地冷却或不可用状态。

### 修复渲染报错被错误归类为“模型不支持多模态”的问题
- 修复了当 AI 渲染发生其他错误（例如 API 404 响应细节描述）而抛出异常时，由于异常消息中带有 `"vision"` 或 `"image_url"` 关键字，导致客户端 UI 界面（`app_logic.py`）将其误判并显示为“模型不支持多模态输入”引导信息的 Bug。现已增加豁免判定，排除由渲染器（Renderer）抛出的此类异常误匹配。

### 修复 Sakura API 地址更新与失败处理
- 修复 Sakura 翻译器只读取启动时 `SAKURA_API_BASE` 的问题。现在 UI、Web 用户配置或预设中修改 Sakura 地址后，会在翻译任务解析配置时重新应用，避免地址改为 8081 后实际仍连接 8080。
- Sakura 翻译开始时会在日志中输出当前连接地址，连接重试与最终失败日志也会带上地址，便于定位实际请求的端口。
- 修复 Sakura 连接失败达到重试上限后返回原文、导致流程继续渲染的问题。现在会抛出异常并终止任务，避免把原文误当翻译结果。
