# v2.2.9 更新日志

发布日期：2026-05-28

## ✨ 新功能

### Qt UI：替换规则编辑器自动保存

- 替换规则编辑器移除手动「保存」按钮，表格编辑和原始 YAML 编辑都会在修改后自动保存。
- 原始 YAML 编辑模式下语法错误时不会写入文件，并在状态栏提示「修改尚未保存」，避免错误内容覆盖当前可用规则。
- 新增「恢复默认」按钮，可一键用程序内置的默认替换规则模板重置 `examples/text_replacements.yaml`。
- 恢复默认与自动保存都会同步清理替换规则缓存，后续渲染会读取最新规则。

## ⚡ 优化

- 优化了编辑器加载和渲染速度。
- 编辑器 Ctrl+Q 导出后立刻按 D 切图不再反复弹「未保存的编辑」对话框：导出是异步任务，原本要等后端渲染完成才刷新快照，连按切图会把多个对话框堆到后台。改为提交异步任务前就同步打一次快照，连续切图不再触发误报。

## 🐛 修复

- 修复了在编辑器中添加文件或文件夹时没有进行自然排序的问题。现在导入包含数字命名的文件（如 `1.jpg`、`2.jpg`、`10.jpg`）或子目录时，会正确遵循数字递增规则进行排序，而不再因为字符串比对产生乱序。
- 修复并发流水线中翻译过滤文本块后触发重做修复时，修复线程可能过早退出导致任务卡住的问题。
- 修复了排版引擎因为使用替换规则计算宽度导致 `translation_raw` 数据被“污染”的问题。现在通过差异对比算法（LCS），在确保换行断句极其精准的同时，还能将排版标签 `[BR]` 与 `<H>` 完美映射回未替换的原始文本中，确保编辑器看到的始终是原汁原味的标点。
- 修复靠近图片边缘的文本在导出渲染时被强行压回画布内，导致文字变形、主流程结果与编辑器预览不一致的问题。现在超出画布的部分会自然裁切，不再改变文本框几何。
- 修复编辑器导出时仍会在渲染层二次应用 `text_replacements.yaml` 的问题。导出临时 JSON 现在会在图片节点记录 `skip_text_replacements`，避免替换规则增删字符后导出内容或宽度与编辑器预览不一致。
- 移除已废弃的 `rich_text` 渲染分支，统一使用普通 `translation`、`[BR]` 与 `<H>` 路径，减少预览和导出的文本处理差异。
- 修复开启「膨胀不超过气泡蒙版」后，文本行保护区仍额外外扩 20px 导致气泡外蒙版被保留的问题；现在只保护原始 `region.lines` 最小外接矩形内已有的蒙版。
