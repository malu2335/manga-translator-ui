# v2.2.9 更新日志

发布日期：2026-05-28

## ✨ 新功能

### API 管理

- 新增 API 通道轮询/故障切换，支持 OpenAI/Gemini 在翻译、OCR、上色、渲染中配置多组 Key。
- API 密钥输入框默认隐藏，可在输入框内点击图标显示或隐藏内容。

### Qt UI：界面精细化现代化重构

- **无边框分段控制滑块标签（Segmented Control Tabs）**：移除全部 `QTabWidget` 面板的外边框，将标签栏重构为现代的分段式胶囊按钮（未选中时背景透明，选中/悬停时展示圆角主题高亮 pill 状态），并应用到全局、编辑器、提示词分类及文本过滤编辑器中。
- **对称悬浮导航胶囊（Floating Sidebar Button Pills）**：重写左侧侧边栏按钮样式，移除硬质边框与左侧指示条，使其成为圆角对称悬浮式胶囊按钮，视觉观感更高级、更轻量。
- **全局字体粗细柔化（Typography Softening）**：移除界面中所有沉重刺眼的超粗体（`700`/`800` 粗细），将标题、小标题、列表头及弹窗按钮全局柔化为半粗体（`600`）或中等粗细（`500`），让排版呼吸感更强、更清爽。

### Qt UI：字体管理与预览交互重新设计

- **自定义文本即时预览**：在字体管理页面新增预览文本输入框，支持输入任意中文、日文、英文等字符组合进行即时测试，支持一键清空。
- **滑块控制字号（12pt - 64pt）**：新增预览字号滑块及数字化字号指示标签，拉动滑块可无缝缩放下方所有测试文本大小。
- **多行比例样张展示**：默认状态下分别以 `0.75x` 比例渲染英文字符串、`1.1x` 比例渲染中日韩经典测试句、`1.6x` 比例渲染英文 Pangram 短句。输入自定义文本时，各行同步以大、中、小三个比例展示您的文本，以评估不同字号下的渲染表现。
- **无界滚动预览容器**：将所有预览样张组件嵌入滚动区域，防止字号过大或窗口缩小时发生界面截断。

### Qt UI：开关（Toggle Switch）色彩柔化

- 修复了开关组件在进行颜色混合时丢失透明度通道，导致半透明主题色被强制解析为不透明的「纯黑」（关闭状态）和「饱和纯蓝」（开启状态）的视觉 Bug。
- 引入了背景色融合机制（`_blend_with_background`），将轨道半透明主题色与底层设置卡片背景色实时重叠混合，在浅色模式下实现极具高级感的浅柔灰色（关闭）和马卡龙淡蓝色（开启），在深色模式下则呈现软暗灰色和深靛蓝色，完美融入整体现代化 UI，消除视觉刺眼感。

### Qt UI：替换规则编辑器自动保存

- 替换规则编辑器移除手动「保存」按钮，表格编辑和原始 YAML 编辑都会在修改后自动保存。
- 原始 YAML 编辑模式下语法错误时不会写入文件，并在状态栏提示「修改尚未保存」，避免错误内容覆盖当前可用规则。
- 新增「恢复默认」按钮，可一键用程序内置的默认替换规则模板重置 `examples/text_replacements.yaml`。
- 恢复默认与自动保存都会同步清理替换规则缓存，后续渲染会读取最新规则。

## ⚡ 优化

- 默认 `paddleocr` 识别模型升级为官方 `PP-OCRv6_medium_rec` ONNX 模型，并改为读取 `inference.yml` 内置字典，提升通用多语言 OCR 识别覆盖；`paddleocr_korean` / `paddleocr_latin` / `paddleocr_thai` 专用模型保持原有 PP-OCRv5 版本。
- `paddleocr_vl` OCR 模型升级为官方 `PaddleOCR-VL-1.6`，并改为从 ModelScope 下载到本地 `models/ocr/PaddleOCR-VL-1.6` 目录，避免继续依赖旧版 1.5 私有归档包。
- 优化了编辑器加载和渲染速度。
- 编辑器 Ctrl+Q 导出后立刻按 D 切图不再反复弹「未保存的编辑」对话框：导出是异步任务，原本要等后端渲染完成才刷新快照，连按切图会把多个对话框堆到后台。改为提交异步任务前就同步打一次快照，连续切图不再触发误报。

## 🐛 修复

- 修复 OCR 模型下载时将合法小体积配置文件（如 `generation_config.json`、`inference.yml`）误判为 404 页的问题。
- 修复了在编辑器中添加文件或文件夹时没有进行自然排序的问题。现在导入包含数字命名的文件（如 `1.jpg`、`2.jpg`、`10.jpg`）或子目录时，会正确遵循数字递增规则进行排序，而不再因为字符串比对产生乱序。
- 修复并发流水线中翻译过滤文本块后触发重做修复时，修复线程可能过早退出导致任务卡住的问题。
- 修复了排版引擎因为使用替换规则计算宽度导致 `translation_raw` 数据被“污染”的问题。现在通过差异对比算法（LCS），在确保换行断句极其精准的同时，还能将排版标签 `[BR]` 与 `<H>` 完美映射回未替换的原始文本中，确保编辑器看到的始终是原汁原味的标点。
- 修复靠近图片边缘的文本在导出渲染时被强行压回画布内，导致文字变形、主流程结果与编辑器预览不一致的问题。现在超出画布的部分会自然裁切，不再改变文本框几何。
- 修复编辑器导出时仍会在渲染层二次应用 `text_replacements.yaml` 的问题。导出临时 JSON 现在会在图片节点记录 `skip_text_replacements`，避免替换规则增删字符后导出内容或宽度与编辑器预览不一致。
- 移除已废弃的 `rich_text` 渲染分支，统一使用普通 `translation`、`[BR]` 与 `<H>` 路径，减少预览和导出的文本处理差异。
- 修复开启「膨胀不超过气泡蒙版」后，文本行保护区仍额外外扩 20px 导致气泡外蒙版被保留的问题；现在只保护原始 `region.lines` 最小外接矩形内已有的蒙版。
